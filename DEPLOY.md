# Deploy — aleph01.com (VPS / Plesk)

The site is a **static Vite build**. Source lives in this repo; the servable
output is generated by `npm run build` into `dist/`. The live site is served by
**nginx from a static document root** — no Node process required.

The one rule that makes or breaks a deploy: **never serve the repo root
directly.** The source `index.html` loads `/src/main.jsx` (a dev entry) and only
works *after* a build. The web root must point at the built `dist/`, not at the
raw source.

---

## Recommended — Plesk Git → build → static docroot

This matches the existing Plesk setup (Git repo deploys `main` automatically to
`httpdocs`). Two changes turn the manual flow into a fully automatic one.

**1. Add the build step.** In **Domains → aleph01.com → Git →** repo
**Settings** (the sliders icon), under **Additional deployment actions**, put:

```bash
npm ci --include=dev
npm run build
```

`--include=dev` is required: Vite lives in `devDependencies`, and if Plesk runs
with `NODE_ENV=production` a plain `npm ci` would skip it and the build would
fail with `vite: not found`.

**2. Point the web root at the build output.** In **Domains → aleph01.com →
Hosting & DNS → Hosting settings**, set **Document root** from `httpdocs` to:

```
httpdocs/dist
```

Now the flow is: push to `main` → Plesk pulls → runs the build → nginx serves
`httpdocs/dist`. No manual copying, ever.

### First-time order (avoids a broken window)

1. Add the build step (change 1).
2. Click **Deploy now** once — this creates `httpdocs/dist`.
3. Change the Document root to `httpdocs/dist` (change 2).
4. Verify the site loads, then you're done — every future push auto-deploys.

Node version must be **18+** (Vite 5 / three.js). Set it in **Node.js** or the
hosting settings if the build errors on an old Node.

## Alternative — Plesk Node.js app (`start.js`)

Only needed if you later want real server-side logic; overkill for a static
page. The repo ships `start.js` (a zero-dependency static server for `dist/`)
and `plesk-deploy.sh` for this route:

1. **Domains → aleph01.com → Node.js**: Application Startup File = `start.js`,
   Application Root = repo dir, mode `production`, Node 18+.
2. Git **Additional deployment actions**: `bash plesk-deploy.sh`
   (runs `npm ci --include=dev` → `npm run build` → `touch tmp/restart.txt`,
   which tells Passenger to restart).

Passenger sets `PORT`; `start.js` reads it and serves `dist/` behind nginx.

---

## Manual / one-off deploy

```bash
git pull
npm ci --include=dev
npm run build
# static: publish dist/ to the web root (or set docroot to dist)
# node:   node start.js   # serves dist/ on $PORT (default 3000)
```

---

## Notes

- `dist/` is **git-ignored** — always rebuilt on the server, never committed.
- `start.js` caches hashed assets (`immutable`, 1 year) and keeps `index.html`
  `no-cache`; nginx does the same when serving `dist/` statically, so a new
  deploy shows up immediately without stale HTML.
- Hiding/showing page sections is done in `src/components/Content.jsx`
  (`defaultSections`) — just edit and rebuild.
